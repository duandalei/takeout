# 领域文档

工程相关 skill 在探索代码库时，如何消费本仓库的领域文档。

## 探索前，先读取这些

- **`CONTEXT.md`** —— 仓库根目录下
- **`docs/adr/`** —— 读取与当前工作相关的 ADR
- 如果上述文件不存在，**静默跳过**。不要报告缺失，也不要建议立即创建。`/grill-with-docs` skill 会在需要时按需生成。

## 文件结构

本仓库为单上下文结构：

```
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-event-sourced-orders.md
│   └── 0002-postgres-for-write-model.md
└── src/
```

## 使用术语表中的词汇

当你的输出涉及领域概念时（issue 标题、重构提案、假设、测试名称），使用 `CONTEXT.md` 中定义的术语。不要偏离到术语表明确避免的同义词。

如果你需要的概念在术语表中还不存在，这是一个信号 —— 要么你使用了项目不用的语言（重新考虑），要么确实存在缺口（记录下来供 `/grill-with-docs` 处理）。

## 标记 ADR 冲突

如果你的输出与已有 ADR 存在冲突，请明确指出，而不是默默覆盖：

> _与 ADR-0007（事件溯源订单）冲突 —— 但值得重新讨论，因为……_
