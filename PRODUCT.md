# Product

## Register

product

## Users

三种角色共用同一套界面：

- **顾客** — 浏览商家和菜单、下单、查看订单状态、评价已完成订单。场景：想点外卖的普通用户，需要快速找到想吃的、完成下单、跟踪配送进度。
- **商家** — 开店/编辑店铺信息、管理菜品分类和菜品、处理订单（接单→备餐→备好待取）。场景：需要高效处理订单流，管理菜单。
- **骑手** — 查看待接配送订单、接单、标记取餐/送达。场景：需要快速找到可接订单、完成配送任务。

数据库课程设计项目，用户通过预设测试账号体验完整业务流程。

## Product Purpose

Web 外卖订餐平台。演示数据库设计与 Web 应用开发的结合：7 张数据表支撑完整的订单生命周期（浏览→下单→接单→配送→评价），3 种角色协作完成外卖业务闭环。

## Brand Personality

干净、高效、可靠。专业工具风格 — 不花哨、不浮夸，每个设计元素服务于任务完成。像一把好用的工具：你知道它在哪、它做什么、用了就不会出错。

## Anti-references

- 不过度装饰 — 不需要渐变、阴影堆叠、动画炫技
- 不模仿消费级外卖 App 的营销风格（红包弹窗、促销横幅、花哨的启动画面）
- 保持课程项目的克制 — 不一味追求"看起来像商业产品"

## Design Principles

1. **功能优先** — 每个设计决定都服务于任务完成；没有纯装饰元素。按钮说清楚会发生什么，状态一眼可辨，下一步操作显而易见。
2. **清晰直白** — 信息层级让下一步行动不言自明。不用图标猜谜，不用省略号制造悬念。
3. **一致可靠** — 三种角色下的交互模式统一。同样的状态标签在所有页面长得一样，同样的按钮行为一致。可预测就是可用。
4. **轻量克制** — Bootstrap 5 为基础，自定义 CSS 只补不足。不引入额外前端框架或构建工具。保持课程项目可维护、可理解。

## Accessibility & Inclusion

课程项目，无正式 WCAG 合规要求。基本可读性保证：正文与背景保持合理对比度，状态信息不单独依赖颜色（配合文字标签）。
